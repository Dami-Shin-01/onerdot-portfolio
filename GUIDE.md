# On a DoT 웹사이트 콘텐츠 관리 가이드

> 이 가이드는 코딩 지식 없이 웹사이트 콘텐츠를 직접 수정할 수 있도록 작성되었습니다.

---

## 1. 블로그 글 작성하기

### 새 글 작성

1. GitHub에 로그인합니다
2. 저장소의 `posts/` 폴더로 이동합니다
3. 오른쪽 상단 **Add file > Create new file** 클릭
4. 파일 이름: `글-제목.mdx` (예: `ai-basics.mdx`)
5. 아래 템플릿을 복사해서 붙여넣고, 내용을 수정합니다

```mdx
---
title: "글 제목을 여기에"
date: "2026-04-07"
tags: ["AI 기초", "비전공자"]
summary: "이 글을 한 줄로 요약하면?"
published: true
---

여기부터 본문을 작성하세요.

## 소제목은 이렇게

본문 내용을 이어서 작성합니다.

**굵은 글씨**는 별표 두 개로 감싸면 됩니다.

- 목록은 이렇게
- 대시로 시작하면 됩니다
```

6. 하단 **Commit changes** 버튼을 클릭합니다
7. 약 1-2분 후 사이트에 자동 반영됩니다

### 기존 글 수정

1. `posts/` 폴더에서 수정할 `.mdx` 파일 클릭
2. 오른쪽 상단 연필 아이콘(Edit) 클릭
3. 내용 수정 후 **Commit changes** 클릭

### 글 비공개 처리

글 상단의 `published: true`를 `published: false`로 바꾸면 사이트에서 숨겨집니다.

---

## 2. 블로그 이미지 추가하기

1. `public/images/posts/` 폴더로 이동
2. **Add file > Upload files** 클릭
3. 이미지 파일을 업로드합니다
4. 블로그 글에서 아래처럼 사용합니다:

```mdx
![이미지 설명](/images/posts/파일명.webp)
```

> 이미지는 가능하면 `.webp` 또는 `.jpg` 형식을 사용해주세요.
> 파일 크기는 500KB 이하를 권장합니다.

---

## 3. 강의 이력 수정하기

강의 이력 데이터는 `content/lectures.json` 파일에 있습니다.

### 숫자 수정 (누적 강의 횟수 등)

`stats` 부분의 숫자를 바꿔주세요:

```json
"stats": {
  "totalLectures": 150,      ← 누적 강의 횟수
  "organizations": 40,       ← 출강 기관 수
  "totalParticipants": 3000  ← 누적 수강 인원
}
```

### 이력 추가

`timeline` 배열에 새 항목을 추가합니다. 가장 최근 연도가 위에 옵니다:

```json
{
  "year": "2026",
  "items": [
    { "org": "기관명", "topic": "강의 주제" },
    { "org": "기관명", "topic": "강의 주제" }
  ]
}
```

### 올바른 예시

```json
{
  "year": "2026",
  "items": [
    { "org": "삼성전자", "topic": "생성형 AI 활용 워크숍" },
    { "org": "서울시청", "topic": "AI 리터러시 교육" }
  ]
}
```

### 잘못된 예시 (자주 하는 실수)

```json
// 실수 1: 마지막 항목 뒤에 쉼표 붙임 (오류 발생)
{ "org": "서울시청", "topic": "AI 리터러시 교육" },  ← 이 쉼표 삭제

// 실수 2: 따옴표 빠뜨림 (오류 발생)
{ "org": 삼성전자, "topic": "워크숍" }  ← "삼성전자"로 수정

// 실수 3: 중괄호/대괄호 짝이 안 맞음 (오류 발생)
{ "org": "삼성전자", "topic": "워크숍"   ← 닫는 } 빠짐
```

> 수정 후 사이트에 오류가 나면, 이전 버전이 자동으로 유지되니 걱정하지 않으셔도 됩니다.

## 4. 문의 폼 (Formspree)

현재 문의 폼은 **Formspree 무료 플랜**(월 50건)으로 운영됩니다.
- 폼으로 들어온 문의는 등록된 이메일로 바로 전달됩니다
- 월 50건 초과 시 문의가 접수되지 않을 수 있습니다
- 문의량이 늘어나면 Formspree 유료 플랜(월 $8, 1000건)으로 전환을 권장합니다
- Formspree 대시보드: https://formspree.io (가입된 계정으로 로그인)

---

## 5. 주의사항

- **수정 가능한 파일**: `posts/` 폴더, `public/images/posts/` 폴더, `content/lectures.json`
- **수정하면 안 되는 파일**: 그 외 모든 파일 (`src/`, `app/`, `components/` 등)
- 수정 후 사이트 반영까지 약 1-2분 소요됩니다
- 빌드 실패 시 이전 배포가 자동 유지됩니다 (안전)
- 문제가 생기면 개발자에게 연락해주세요
